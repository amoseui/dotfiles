# obsidian-history — vault 수정 이력 관리

Obsidian vault 내 .md 파일을 수정할 때마다 두 곳에 수정 이력을 기록한다.

## 왜 이력을 관리하는가

Amos는 Obsidian vault를 개인 지식 관리 시스템으로 사용하고 있다. 파일이 언제, 왜 변경되었는지 추적할 수 있으면 나중에 변경 맥락을 빠르게 파악할 수 있고, 실수로 잘못 고친 부분도 쉽게 찾을 수 있다. 따라서 모든 수정에는 이력이 따라붙어야 한다.

## 이력 기록 위치 (두 곳 모두 기록)

### 1. 파일 하단 `## History` 섹션

수정한 파일의 맨 아래에 수평선(`---`)과 `## History` 섹션을 유지한다.

- 본문 마지막 내용과 History 사이에는 **빈 줄 2개 + `---`** 로 시각적으로 구분한다.
- 파일에 이미 `## History` 섹션이 있으면 헤딩 바로 아래(기존 항목 위)에 새 항목을 삽입한다. 즉 **최신 항목이 항상 맨 위**에 온다.
- 없으면 파일 맨 끝에 빈 줄 2개 + `---` + `## History` 헤딩과 함께 새로 만든다.
- 기존 History 항목은 절대 수정하거나 삭제하지 않는다.

형식 (시간 포함):
```
## History
- YYYY-MM-DD HH:MM 변경 내용을 한줄로 간결하게 요약
```

배치 예시 (최신이 위):
```

---
## History
- 2026-04-13 14:30 저녁 루틴 체크리스트 항목 수정
- 2026-04-13 06:38 아침 브리핑 섹션에 Google Calendar 일정 자동 채움
```

### 2. `wiki/log.md` 통합 로그

vault 루트 기준 `wiki/log.md` 파일에도 변경 사항을 기록한다.

- 파일 경로: `/Users/amoseui/Obsidian/amoseui/amoseui/wiki/log.md`
- `# Change Log` 헤딩 바로 아래(기존 항목 위)에 새 줄을 삽입한다. 즉 **최신 항목이 항상 맨 위**에 온다.
- **예외**: `wiki/log.md` 자체를 수정하는 경우에는 log.md에 중복 기록하지 않는다.

형식 (시간 포함):
```
- YYYY-MM-DD HH:MM `vault 루트 기준 상대 경로` — 변경 내용 요약
```

예시 (최신이 위):
```
# Change Log

- 2026-04-13 14:30 `Templates/template-retrospective-1-daily.md` — 저녁 루틴 항목 수정
- 2026-04-13 06:38 `Retrospective/1. Daily/2026-04-13.md` — 아침 브리핑 자동 업데이트
```

## 변경 내용 요약 작성 가이드

- 한국어로 작성한다.
- 한 줄로 간결하게, 무엇이 왜 바뀌었는지 핵심만 담는다.
- 여러 변경을 한 번에 했다면 쉼표로 구분하여 한 줄에 담는다.
- 자동 작업(아침 브리핑, 일일 회고 등)의 경우 "자동 업데이트" 등의 표현을 포함한다.

좋은 예: `아침 브리핑 자동 업데이트 (캘린더 일정, 이메일 요약)`
나쁜 예: `파일을 수정했습니다` (너무 모호함)

## 적용 순서

파일을 수정할 때 다음 순서로 진행한다:

1. 본문 내용을 먼저 수정한다.
2. 해당 파일의 `## History` 섹션에 항목을 추가한다.
3. `wiki/log.md`에 항목을 추가한다 (log.md 자체 수정이 아닌 경우).

## 적용 범위

- **대상**: `/Users/amoseui/Obsidian/amoseui/amoseui/` 하위의 모든 `.md` 파일
- **제외**: `wiki/log.md`를 수정할 때는 log.md 자체에 대한 로그를 log.md에 중복 기록하지 않음 (파일 내 History 섹션은 기록함)
- **제외**: 새 파일을 처음 생성(Write)하는 경우, History의 첫 항목은 "최초 생성"으로 기록

## 날짜 확인

오늘 날짜와 현재 시각은 시스템 환경에서 확인한다. bash로 `date +"%Y-%m-%d %H:%M"`을 실행하여 `YYYY-MM-DD HH:MM` 형식으로 확인한다. 시간대는 Asia/Seoul (KST) 기준이다.
